# PDF Export Instructions

Use this guide when turning `docs/report.md` into the final submission PDF.

## Steps

1. Open the Google Docs report template provided for the hackathon.
2. Copy the content from `docs/report.md` into the template.
3. Insert the main screenshots from `public/screenshots`:
   - `home-hero.png`
   - `triage-report.png`
   - `dashboard.png`
   - `benchmark-lab-summary.png`
4. Keep these as the main report figures:
   - Figure 1: CivicGuard Africa workflow and demo entry point
   - Figure 2: Generated triage report for a demo election incident
   - Figure 3: Dashboard overview of civic risk signals
   - Figure 4: Benchmark Lab for multilingual AI safety evaluation
5. Add appendix screenshots only if there is space:
   - `submit-demo-filled.png`
   - `community-review.png`
   - `benchmark-lab-full.png`
   - `about.png`
6. Check page breaks, image size, and table readability.
7. Confirm the GitHub repository link is present:
   - https://github.com/josuekb09/civicguard-africa
8. Confirm the deployed demo link is present:
   - https://civic-guard-africa.vercel.app/
9. Export the final document as PDF.
10. Submit the PDF with the GitHub repository link and deployed demo link.

## Final Visual Check

Before submitting, visually confirm that:

- The report title, author, and affiliation are correct.
- The abstract fits on the first page cleanly.
- Screenshots are readable and not stretched.
- Tables do not overflow the page.
- References contain real source links and no placeholder URL text.
- The LLM usage statement and limitations are included.
