# Screenshots Needed

Screenshots can be generated automatically.

Run the app:

```bash
npm run dev
```

In another terminal, capture the screenshots:

```bash
npm run capture:screenshots
```

The files will be saved in:

```text
public/screenshots
```

## 1. Home Page Hero And Judge Demo Flow

Why useful: Shows the project name, one-line pitch, main CTA buttons, and the judging workflow. This is the best opening screenshot for the report or final presentation.

## 2. Submit Page With Loaded Demo Case

Why useful: Demonstrates the structured intake form and the safe demo election case. It proves the workflow begins with contextual data rather than vague text input.

## 3. Triage Report Page After Generating Demo Case

Why useful: Shows the core output of the tool: incident ID, risk level, harm category, confidence estimate, explanation, checklist, reviewer next steps, and safety disclaimer.

## 4. Community Review Queue Showing Submitted Incident

Why useful: Shows how the MVP simulates human verification with filters, review status, and reviewer notes.

## 5. Dashboard With Charts And Metrics

Why useful: Shows civic risk monitoring: total incidents, high-risk incidents, pending reviews, verified cases, charts, and high-risk table.

## 6. Benchmark Lab With Summary Cards, Charts, And Table

Why useful: Shows the Evaluation & Benchmarking contribution, including multilingual coverage, harm categories, scoring labels, charts, and exportable benchmark data.

## 7. About Page If Space Allows

Why useful: Shows the project framing, target users, hackathon alignment, MVP scope, roadmap, and safety/ethics note in one judge-friendly page.
